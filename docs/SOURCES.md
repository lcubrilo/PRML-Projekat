# Sources (grounding the claims in CLAUDE.md)

Light verification pass - confirming the load-bearing claims have real sources. Not exhaustive.

## Predictability ceiling (~65%)
- Multiple UFC winner-prediction models **cluster around ~65%** accuracy (Kappa < 0.6). This is the central tendency, **not a hard wall**:
  - careful ensembles reach **~67–68%** (e.g. 67.6% acc, 0.727 AUC, 0.598 log-loss on held-out data);
  - one soft-voting ensemble (XGBoost+LightGBM+LogReg, Bayesian tuning, **SMOTE**, time-based split) claims **79.25%** acc / 88.67% AUC - ⚠️ treat with leakage/over-optimism scrutiny given the SMOTE + the gap to the pack.
- **Honest framing for the report:** typical ~63–67%; sophisticated work high-60s; >75% claims warrant a leakage audit. The "~65% ceiling" narrative is defensible as *typical*, not absolute.

## Feature-engineering norm
- The ~**63 differential ("red − blue") features** approach (stance compatibility, last-5 form, win-streak/age/reach diffs, striking/TD stats) is the standard representation - confirms our difference-feature plan.

## Related public projects (how ours differs)
Light scan - the genre is **winner-prediction-heavy** (logistic/RF/XGBoost on the same ufcstats lineage): rezan21, calicartels, Carson Martin's full-stack app, the ACM 2024 paper. Method-of-victory **is** done occasionally (Carson Martin predicts method; comparative studies use RF/GBM/SVM), so the *task* isn't unique. **Our differentiation is the combination:** (1) everything from scratch, (2) explicit leakage audit + chronological split, (3) the red-corner confound as controlled EDA, (4) benchmarking vs the **per-method market odds** (rare). None of these are common in the public pile.

## Kaggle / dataset community notes
- Documented gotcha for `rajeevw/ufcdata`: **many NaNs, mostly from debut fighters** (each row is the career average *excluding* the current fight, so first fights have no priors) - exactly the leakage-safe design, not a bug. Many **categorical** columns also need encoding.
- `mdabbert` = a merge of the public UFC sets (Warrier stats + odds + rankings), generated by the `shortlikeafox/ultimate_ufc_dataset` scraper; "constant work in progress," `upcoming.csv` carries the next event.
- No widespread **leakage complaints** on the pre-aggregated files; the leakage trap only bites if you build features off raw in-fight stats (komaksym `stats_raw`).
